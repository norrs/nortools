## NorTools v0.0.260220002

This release focuses on major web UX improvements, DNS tooling enhancements, and desktop/runtime quality updates.

### Highlights

- Reworked multiple tools from raw JSON output into readable report-style interfaces with dedicated `JSON` tabs.
- Expanded DNS Lookup with built-in `CAA` and `ANY` support plus custom record types.
- Added a new in-app DNS Record Types help page with RFC references.
- Improved HTTPS certificate visibility with human-readable expiry timing.

### Web UI Improvements

- Added cleaner report views (with `Report` + `JSON` tabs) for:
  - Subnet Calculator
  - Password Generator
  - HTTP Check
  - TCP Check
  - Reverse DNS Lookup
  - DNS Lookup
- Blacklist Check now presents all checked DNSBLs in one unified, easier-to-scan result table.
- WHOIS Lookup now opens `Raw WHOIS` by default when parsed fields are unavailable.
- Traceroute now hides the map tab when geo data is missing and clearly explains map unavailability.
- Interfaces & Routing now supports click-through navigation from summary/route rows to the selected interface card.
- HTTPS/SSL view now shows certificate expiry in plain language (for example: `Expires in N days`, `Expired N days ago`).

### DNS & Help Content

- DNS Lookup:
  - Added default `CAA` record type option.
  - Added default `ANY` record type option.
  - Added custom DNS record type input.
- Added new help page: `DNS Record Types Help` with categorized record descriptions and RFC links.
- Linked DNS help directly from the DNS Lookup page and from the sidebar help section.

### Desktop / Runtime / Tooling

- Migrated web runtime to Javalin 7.
- About-page links now open in the native browser.
- Improved native/runtime compatibility and build pipeline pieces:
  - Switched native builds to GraalVM Community 25.0.2.
  - Added macOS arm64 native run convenience target.
  - Bazel docs resource jar fixes and cleanup.
- Added developer setup documentation.

### Notes

- This release includes both user-facing improvements and internal build/runtime hardening.
- No manual migration steps are required for standard users.
